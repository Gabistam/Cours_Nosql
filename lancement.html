⚙️ Deux processus, une seule app

🖥️
Serveur — Express
Port 3001 · Node.js

Le serveur expose une API REST. Il reçoit des requêtes HTTP, interroge les bases de données, et renvoie du JSON. Il ne sait rien de React.

  • Répond sur http://localhost:3001
  • Toutes les routes commencent par /api/
  • Sert aussi les images sur /img/
  • Lance avec npm run dev dans server/
⚛️
Client — React + Vite
Port 5173 · navigateur

Le client est une SPA (Single Page Application). Vite sert les fichiers React en développement et proxy les appels API vers Express.

  • Répond sur http://localhost:5173
  • C'est l'URL que l'utilisateur ouvre
  • Les appels /api/* sont redirigés vers :3001
  • Lance avec npm run dev dans client/

🔀 Le proxy Vite — pourquoi c'est essentiel

🔀
Comment React parle à Express
client/vite.config.js

Sans proxy, quand React fait fetch('/api/products') depuis le port 5173, le navigateur cherche localhost:5173/api/products — introuvable. Le proxy Vite intercepte toutes les requêtes /api/* et /img/* et les redirige vers le port 3001 de manière transparente.

⚛️ React
localhost:5173
→
🔀 Proxy Vite
/api/* /img/*
intercepte les requêtes
→
🖥️ Express
localhost:3001
javascript · client/vite.config.js
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  server: {
    port: 5173,
    proxy: {
      '/api': { target: 'http://localhost:3001', changeOrigin: true },
      '/img': { target: 'http://localhost:3001', changeOrigin: true },
    },
  },
})
En production, ce proxy n'existe plus. À la place, un vrai reverse proxy (Nginx, Caddy) ou un hébergement comme Vercel/Render gère la redirection. Pour le cours, Vite suffit.

🩺 Le point de contrôle /api/health

🩺
Vérifier que tout tourne

Avant de coder quoi que ce soit, vérifie toujours ce endpoint. Il indique si le serveur Express est en vie et si la base de données est connectée.

javascript · server/app.js (extrait)
app.get('/api/health', (req, res) => {
  const mongoose = require('mongoose')
  res.json({
    success: true,
    data: {
      status: 'ok',
      phase: 'mongodb',
      db: mongoose.connection.readyState === 1 ? 'connecté' : 'déconnecté',
    },
  })
})

Installation du projet

Trois étapes : cloner le dépôt, installer les dépendances des deux sous-projets, configurer les variables d'environnement. Compte environ 5 minutes.

✅ Prérequis

⚙️
Ce dont tu as besoin avant de commencer

Logiciels obligatoires

  • Node.js 18+ — node --version doit afficher v18.x ou plus
  • Git — git --version
  • Un terminal — PowerShell, Git Bash, ou Terminal VSCode
  • VSCode (recommandé) avec l'extension ES7 React Snippets

Pour les phases suivantes (pas encore)

  • Docker Desktop — pour Redis, Cassandra, Neo4j
  • Un compte MongoDB Atlas (gratuit) — pour Phase 1
bash · vérification rapide
node --version # → v20.x.x ou v22.x.x
npm --version # → 9.x.x ou plus
git --version # → git version 2.x.x
Node 16 ou moins ? Certaines dépendances (Vite 5, Mongoose 9) nécessitent Node 18 minimum. Installe la dernière LTS depuis nodejs.org.

Étape 3 — Configurer le .env

🔐
Variables d'environnement
server/.env (à créer depuis server/.env.example)

Le fichier .env contient les secrets et la config locale. Il n'est jamais commité (il est dans le .gitignore). Tu le crées une fois sur ta machine.

bash · créer le .env
# Depuis le dossier server/
cp ../.env.example .env
# Puis édite le fichier .env avec ton éditeur
.env · server/.env
# ── Serveur ────────────────────────────────────────
PORT=3001
NODE_ENV=development

# ── MongoDB Atlas (Phase 1) ──────────────────────────
MONGODB_URI=mongodb+srv://<user>:<password>@<cluster>.mongodb.net/sportshopy

# ── Redis (Phase 2) ──────────────────────────────────
# REDIS_URL=redis://localhost:6379

# ── Cassandra (Phase 3) ──────────────────────────────
# CASSANDRA_CONTACT_POINTS=localhost
# CASSANDRA_KEYSPACE=sportshopy
🚫 Ne jamais committer le .env ! Il contient ton mot de passe Atlas. Vérifie avec git status — le fichier .env doit apparaître en rouge (non suivi) ou ne pas apparaître du tout.
Pour l'instant, tu peux laisser MONGODB_URI vide ou avec une valeur factice. Le serveur démarrera mais affichera une erreur de connexion — c'est normal pour la Phase 0. On le remplit au cours suivant (MongoDB).

Lancer le projet en dev

Il faut deux terminaux simultanés — un pour le serveur, un pour le client. Les deux doivent tourner en même temps pour que l'application fonctionne.

Vérifier le hot-reload
Ouvre un composant React (ex: client/src/components/common/Header.jsx), change un texte, sauvegarde. Le navigateur doit se mettre à jour instantanément sans F5 grâce au HMR (Hot Module Replacement) de Vite.
